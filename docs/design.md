# 週報 設計書 v1.0 — 割安×モメンタム複合スクリーナー

「三本の矢」設計書v1.0の第1の矢(モメンタム)を、API料金と運用事故リスクを踏まえて
**割安×モメンタム複合スクリーナー + 月1回の手動S株発注**に置き換えたもの。

## スコアリング仕様

### 1. ユニバースフィルター(順に適用・各段階の残数をレポートに記録)

1. ユニバース1000銘柄(JPX上場・時価総額優先で構築済み)
2. `include_for_collection = 1`
3. `low_liquidity_candidate = 0`(流動性懸念銘柄を除外)
4. `is_financial = 0`(銀行・保険は営業CF・自己資本比率の意味が異なるためv1では除外)
5. 株価が取得できている(直近7日以内の終値がある)
6. コア4指標(益回り・予想益回り・簿価利回り・営業CF利回り)のうち3つ以上が計算可能
7. クオリティゲート: `trailingEps > 0` かつ `operatingCashflow > 0`(最終黒字・営業CF黒字)
8. モメンタムフィルター: 12-1リターンが残存銘柄の下位20%でない

### 2. データソース(v1.1で変更)

- **バリュー指標の一次ソースはYahoo Finance(yfinance)**。1株あたり値(EPS・BPS)÷株価で
  計算し、財務諸表の単位問題を回避する
- EDINETスナップショットは**EPS検算(第二ソース)専用**。v1.0でEDINET抽出値を一次ソースに
  した結果、(1)equityに総資産が混入、(2)千円/米ドル建て報告銘柄の単位誤り、
  (3)半期報告書のPLが年度として混入、の3つの系統的問題をYahoo実値照合で検出したため降格した。
  EDINET側の抽出品質が改善したら一次ソースへの昇格を再検討する
- 両ソースのEPS一致率(±25%)を毎号レポートに記録する

### 3. バリュー複合スコア(すべて「利回り」方向・高い=割安)

| 指標 | 定義(Yahoo Financeフィールド) |
|---|---|
| 益回り | trailingEps ÷ 株価 |
| 予想益回り | forwardEps ÷ 株価 |
| 簿価利回り | bookValue ÷ 株価(=1/PBR) |
| 営業CF利回り | operatingCashflow ÷ marketCap |
| 配当利回り | dividendYield(無配・欠損は0%扱い) |

各指標を1%/99%点でウィンザライズ → 横断パーセンタイルランク(0〜100) → 利用可能な指標の平均が複合スコア。

### 4. モメンタム(バリュートラップ除け)

- 12-1リターン = 終値(21営業日前) ÷ 終値(252営業日前) − 1
- 履歴が252営業日未満の銘柄はこのフィルターを通さず除外
- 複合スコア計算後、**下位20%を除外**(買い推奨はしないが下落ナイフを掴まない)

根拠: バリューとモメンタムの併用(Asness, Moskowitz & Pedersen 2013 "Value and Momentum
Everywhere")。純バリューはバリュートラップで報われるまでが長い。

### 5. 絶対バリュー判定(相対ランキングと独立・v1.2で追加)

相対ランキングは「市場全体が割高でも誰かが1位になる」ため、**水準ベースの判定**を併記し、
買うかどうかはこちらで決める。**買いは義務ではない。候補ゼロの号は現金温存が判断。**

| 判定 | 意味 | 条件 |
|---|---|---|
| S | 非常に割安・買い妙味大 | 4基準すべて、または3基準+予想PER8倍以下 |
| A | 十分割安 | 3基準 |
| B | 割安寄り | 2基準 |
| C | 様子見 | 1基準以下 |

基準(4つ): 予想PER10倍以下(欠損時は実績PER) / PBR0.8倍以下 / 配当利回り3.5%以上 /
営業CF利回り12%以上。

閾値は日本市場の長期中央値(PER14〜16倍・PBR1.2前後・配当2%強)に対して明確な
マージンを置いた固定値。金利レジームが大きく変わったら月次レビューで見直す
(場当たり的変更は禁止 — AGENTS.md)。

### 6. 出力と売買候補

- ランキング上位20銘柄(全指標+判定バッジつき)
- **買い候補 = S判定のみ**を毎号提示。株数は単元(`portfolio.json` の lot_size=100株)固定、
  概算金額を併記。何銘柄買うか(買わないか)は資金と相談して人間が決める
- 売り候補は**月次号のみ**: 保有銘柄が複合ランク20位圏外に落ちたら売り候補
  (ヒステリシスで回転コストを抑制)
- 週次号も買い候補(S判定)は表示する — 「非常に割安」は月初を待たず把握したいため

## データ鮮度と限界(正直な記載)

- バリュー指標はYahoo Finance(非公式API)依存。仕様変更・レート制限で欠損が増えたら
  レポートの鮮度パネルに現れる(欠損銘柄はスコア対象から外れるだけで補完しない)
- EDINETスナップショットは検算専用。抽出品質の既知問題(equityへの総資産混入・
  単位誤り・半期PL混入・treasury_shares/配当/forecast_epsのカバレッジ0)が解決するまで
  スコアには使わない
- バリュー複合のヒストリカルバックテストは未実施(ポイントインタイム過去財務が無料で
  入手不可のため)。根拠は学術文献+運用開始後のウォークフォワード記録で代替する
- モメンタム(12-1)は `scripts/backtest_momentum.py` で検証する(現ユニバースでの検証の
  ため生存者バイアスあり。結果は割り引いて読む)

## 期待値(設計書v1.0 §8 を踏襲)

保証は一切ない。バリュー・モメンタムの文献ベースのシャープレシオは0.3〜0.6程度。
悪い年は−10〜15%があり得る。停止条件(AGENTS.md)には必ず従う。
